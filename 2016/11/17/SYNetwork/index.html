SYNetwork

发表于 2016-11-17   |   分类于 iOS   |  

基于AFNetworking的网络请求框架。采用command设计模式，支持按照接口缓存请求数据。

GitHub地址

原理

组成

SYNetwork由一下三个部分组成

  • SYNetwork

    • SYBaseRequest

      所有网络请求 API 的父类

    • SYNetworkConfig

      环境配置 包括网络请求的 BaseUrl 请求超时时间，缓存总量等

    • SYRequestCenter

      所有网络请求统一由这个类发出

    • SYResponse

      网络请求返回值对象

  • SYCache

    • SYCache

      缓存管理

    • SYCacheObject

      缓存的对象

  • Category

    • NSDictionary+SYNetworking

      加密用，在网络请求开始的时候对参数进行加密等，如果需要加密在这个category中惊醒

    • NSString+Encryption

      加密算法，提供md5 盒bas_64

    • NSUrlRequest+SYNetworkingMethods

      动态的给UrlRequest 添加一个请求参数属性

流程图

注意:

  • 缓存的存取 key 是通过参数和url拼接形成的，这里的参数是加密之前的参数
  • 加密的行为是最终发起网络请求的时候就行的
  • 自定义加密算法的时候应该在NSUrlRequest+SYNetworkingMethods类别中进行

使用

首先将SYNetwork导入你的工程。

1、首先导入头文件

2、 在appdelegate中使用SYNetworkConfig配置网络请求的baseurl等

采用command设计模式所以你的网络请求api都应该是一个单独的类

3、创建LoginApi类 .h文件构造方法

.m 文件提供请求的参数和url

4、调用，支持block 和delegate 回调两种方式

如果是delegate方式

应该遵守SYRequestCallBackDelegate协议

并且实现对应方法。
